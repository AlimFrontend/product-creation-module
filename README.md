# Модуль создания карточки товара для маркетплейса

Веб-приложение для создания и отправки карточек товаров в API TableCRM. Одна страница с формой: основная информация, цены, SEO, местоположение, черновики в браузере и подсказки по заполнению (мок «нейросети»).

---

## Как запустить у себя

**Требования:** Node.js 18+, npm.

```bash
# Клонировать репозиторий и зайти в папку
git clone <url-репозитория>
cd marketplace-product

# Установить зависимости
npm install

# Создать файл с переменными окружения (см. ниже)
cp .env.example .env.local
# Отредактировать .env.local — подставить свой токен TableCRM

# Запустить в режиме разработки
npm run dev
```

Открыть в браузере: **http://localhost:3000**

На главной откроется форма создания товара. После заполнения и нажатия «Опубликовать товар» данные отправляются POST-запросом на API TableCRM (массив из одного объекта).

---

## Переменные окружения

В корне проекта создать `.env.local` (по образцу `.env.example`):

| Переменная | Описание |
|------------|----------|
| `NEXT_PUBLIC_API_URL` | Базовый URL API номенклатуры, например: `https://app.tablecrm.com/api/v1/nomenclature/` |
| `NEXT_PUBLIC_API_TOKEN` | Токен доступа к API TableCRM |

Без этих переменных запрос на создание товара выдаст ошибку в интерфейсе.

## Стек

- **Next.js 14** (App Router), **React 18**, **TypeScript**
- **TailwindCSS** — стили
- **React Hook Form** + **Zod** + **@hookform/resolvers** — форма и валидация
- **TanStack Query** — мутации к API
- **next-themes** — тема (поддержка тёмного режима)
- UI в стиле **shadcn**: свои компоненты (Button, Input, Card, Badge, Alert и др.) на базе Radix UI и CVA

Сборка — встроенная в Next.js (не Vite).

---

## Что есть в проекте

- Форма по блокам: основная информация, цены и категории, SEO, местоположение.
- Автосохранение черновика в `localStorage` и восстановление при перезагрузке.
- Автозаполнение полей по названию товара (описания, SEO, код, категория) — мок без внешнего API.
- Кнопки «Сгенерировать из названия», «Сгенерировать SEO», «Подобрать категорию по названию», «Причесать текст» для описаний.
- Валидация по Zod, сообщения об ошибках под полями.
- Превью карточки товара справа (на десктопе).
- Отправка в TableCRM: тело запроса — массив из одного объекта, метод POST.

## Деплой (Vercel)

1. Запушить репозиторий в GitHub.
2. В Vercel: New Project → Import репозитория.
3. В настройках проекта добавить переменные: `NEXT_PUBLIC_API_URL`, `NEXT_PUBLIC_API_TOKEN`.
4. Деплой. Сборка: `npm run build`, выход — Next.js по умолчанию.

После деплоя форма доступна по выданному Vercel URL.
